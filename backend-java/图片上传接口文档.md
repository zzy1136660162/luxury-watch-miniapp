# 图片上传与展示接口文档

## 一、接口概述

### 1.1 接口说明

本文档详细描述了图片上传与图片展示两大核心接口的技术实现细节，供前后端开发人员参考对接。

### 1.2 接口列表

| 接口名称 | 请求URL | 请求方法 | 说明 |
|---------|---------|---------|------|
| 单个图片上传 | `/api/upload/image` | POST | 上传单张图片并返回访问路径 |
| 批量图片上传 | `/api/upload/images` | POST | 批量上传多张图片 |
| 图片访问 | `/images/{year-month}/{filename}` | GET | 通过URL访问已上传的图片 |

### 1.3 服务器信息

| 配置项 | 值 |
|-------|-----|
| 服务器地址 | `http://localhost:8081` |
| 图片存储路径 | `D:/项目图标` |
| 图片访问基础URL | `/images` |
| 是否需要登录 | 否（已加入白名单） |

## 二、单个图片上传接口

### 2.1 接口信息

| 属性 | 值 |
|-----|-----|
| 接口名称 | 单个图片上传 |
| 请求URL | `/api/upload/image` |
| 请求方法 | `POST` |
| Content-Type | `multipart/form-data` |
| 是否需要认证 | 否 |

### 2.2 请求头

| 参数名称 | 数据类型 | 是否必填 | 说明 |
|---------|---------|---------|------|
| 无特殊要求 | - | - | 标准HTTP请求头即可 |

### 2.3 请求参数

| 参数名称 | 参数类型 | 是否必填 | 说明 |
|---------|---------|---------|------|
| file | MultipartFile | **必填** | 上传的 图片文件，支持 jpg、jpeg、png、gif、webp 格式 |

### 2.4 请求示例

#### 使用 cURL

```bash
curl -X POST http://localhost:8081/api/upload/image \
  -H "Content-Type: multipart/form-data" \
  -F "file=@/path/to/your/image.jpg"
```

#### 使用 Postman

1. 请求方法选择：`POST`
2. 请求地址：`http://localhost:8081/api/upload/image`
3. Body 选项卡选择：`form-data`
4. KEY 输入框：`file`
5. VALUE 类型：选择 `File`
6. 选择要上传的图片文件
7. 点击 `Send` 发送请求

#### 使用 JavaScript (Axios)

```javascript
const formData = new FormData();
const fileInput = document.querySelector('input[type="file"]');
formData.append('file', fileInput.files[0]);

axios.post('http://localhost:8081/api/upload/image', formData, {
  headers: {
    'Content-Type': 'multipart/form-data'
  }
}).then(response => {
  console.log('上传成功:', response.data);
}).catch(error => {
  console.error('上传失败:', error);
});
```

#### 使用 Java (OkHttp)

```java
OkHttpClient client = new OkHttpClient();
MediaType mediaType = MediaType.parse("image/jpeg");
RequestBody body = new MultipartBody.Builder()
    .setType(MultipartBody.FORM)
    .addFormDataPart("file", "image.jpg",
        RequestBody.create(mediaType, new File("/path/to/image.jpg")))
    .build();

Request request = new Request.Builder()
    .url("http://localhost:8081/api/upload/image")
    .post(body)
    .build();

try (Response response = client.newCall(request).execute()) {
    System.out.println(response.body().string());
}
```

### 2.5 响应格式

#### 成功响应

| 属性 | 类型 | 说明 |
|-----|------|------|
| code | int | 状态码，200 表示成功 |
| msg | String | 响应消息，"success" 表示成功 |
| data | Object | 图片上传结果数据对象 |

#### data 对象结构

| 属性 | 类型 | 说明 |
|-----|------|------|
| url | String | 图片访问URL路径（相对路径） |
| filename | String | 服务器存储的文件名 |
| size | Long | 文件大小（字节） |
| uploadTime | String | 上传时间（格式：yyyy-MM-ddTHH:mm:ss） |
| originalFilename | String | 原始文件名 |
| fileType | String | 文件扩展名（不含点） |

#### 成功响应示例

```json
{
    "code": 200,
    "msg": "success",
    "data": {
        "url": "/images/2026-03/heroImage_1743401234567_abc123de.jpg",
        "filename": "heroImage_1743401234567_abc123de.jpg",
        "size": 102400,
        "uploadTime": "2026-03-31T14:30:00",
        "originalFilename": "heroImage.jpg",
        "fileType": "jpg"
    }
}
```

#### 错误响应

| 属性 | 类型 | 说明 |
|-----|------|------|
| code | int | 状态码，500 表示失败 |
| msg | String | 错误消息 |
| data | null | 空值 |

#### 错误响应示例

```json
{
    "code": 500,
    "msg": "上传文件不能为空",
    "data": null
}
```

```json
{
    "code": 500,
    "msg": "不支持的文件类型，仅支持：jpg, jpeg, png, gif, webp",
    "data": null
}
```

```json
{
    "code": 500,
    "msg": "文件大小超过限制，最大支持 10MB",
    "data": null
}
```

### 2.6 错误码说明

| 错误码 | 错误消息 | 说明 | 解决方案 |
|-------|---------|------|---------|
| 500 | 上传文件不能为空 | 未选择文件或文件为空 | 确保选择了有效的图片文件 |
| 500 | 不支持的文件类型... | 上传了不支持的图片格式 | 使用支持的格式（jpg, jpeg, png, gif, webp） |
| 500 | 文件大小超过限制... | 上传的文件超过10MB | 压缩图片或降低图片质量 |
| 500 | 文件保存失败... | 磁盘空间不足或权限问题 | 检查D:/项目图标目录的写入权限 |

## 三、批量图片上传接口

### 3.1 接口信息

| 属性 | 值 |
|-----|-----|
| 接口名称 | 批量图片上传 |
| 请求URL | `/api/upload/images` |
| 请求方法 | `POST` |
| Content-Type | `multipart/form-data` |
| 是否需要认证 | 否 |

### 3.2 请求参数

| 参数名称 | 参数类型 | 是否必填 | 说明 |
|---------|---------|---------|------|
| files | MultipartFile[] | **必填** | 多个图片文件数组，支持 jpg、jpeg、png、gif、webp 格式 |

### 3.3 请求示例

#### 使用 cURL

```bash
curl -X POST http://localhost:8081/api/upload/images \
  -H "Content-Type: multipart/form-data" \
  -F "files=@/path/to/image1.jpg" \
  -F "files=@/path/to/image2.jpg" \
  -F "files=@/path/to/image3.png"
```

#### 使用 Postman

1. 请求方法选择：`POST`
2. 请求地址：`http://localhost:8081/api/upload/images`
3. Body 选项卡选择：`form-data`
4. KEY 输入框：`files`
5. VALUE 类型：选择 `File`
6. 可以添加多个 `files` 字段，每个选择一张图片
7. 点击 `Send` 发送请求

#### 使用 JavaScript (Axios)

```javascript
const formData = new FormData();
const fileInput = document.querySelector('input[type="file"]');

// 添加多个文件
for (let i = 0; i < fileInput.files.length; i++) {
    formData.append('files', fileInput.files[i]);
}

axios.post('http://localhost:8081/api/upload/images', formData, {
  headers: {
    'Content-Type': 'multipart/form-data'
  }
}).then(response => {
  console.log('批量上传成功:', response.data);
}).catch(error => {
  console.error('批量上传失败:', error);
});
```

#### data 对象结构

| details | Object[] | 每张图片的详细信息数组 |

### 3.5 错误处理说明

- 如果所有文件都上传失败，返回第一个错误信息
- 如果部分文件上传失败，成功上传的文件仍会返回，并记录警告日志
- 建议在业务逻辑中检查 `count` 字段，确认实际成功数量

## 四、图片访问接口

### 4.1 接口信息

| 属性 | 值 |
|-----|-----|
| 接口名称 | 图片访问 |
| 请求URL | `/images/{year-month}/{filename}` |
| 请求方法 | `GET` |
| 是否需要认证 | 否 |

### 4.2 请求参数

无请求参数，图片路径通过URL路径传递。

### 4.3 请求示例

#### 在浏览器中直接访问

```
http://localhost:8081/images/2026-03/heroImage_1743401234567_abc123de.jpg
```

#### 使用 Postman

1. 请求方法选择：`GET`
2. 请求地址：`http://localhost:8081/images/2026-03/heroImage_1743401234567_abc123de.jpg`
3. 点击 `Send` 发送请求
4. 成功后在响应体中直接显示图片

#### 使用 JavaScript (img标签)

```html
<img src="http://localhost:8081/images/2026-03/heroImage_1743401234567_abc123de.jpg" alt="图片" />
```

#### 使用 JavaScript (CSS背景图)

```css
.div-background {
    background-image: url('http://localhost:8081/images/2026-03/heroImage_1743401234567_abc123de.jpg');
}
```

### 4.4 响应格式

#### 成功响应

- 响应状态码：`200 OK`
- 响应 Content-Type：根据图片格式自动设置
  - JPG/JPEG：`image/jpeg`
  - PNG：`image/png`
  - GIF：`image/gif`
  - WebP：`image/webp`
- 响应体：图片二进制数据

#### 错误响应

| 状态码 | 说明 |
|-------|------|
| 404 Not Found | 图片文件不存在 |
| 500 Internal Server Error | 服务器内部错误或路径配置错误 |

### 4.5 URL路径参数说明

| 参数 | 说明 | 示例 |
|-----|------|------|
| {year-month} | 上传年月（格式：yyyy-MM） | 2026-03 |
| {filename} | 服务器存储的文件名（含扩展名） | heroImage_1743401234567_abc123de.jpg |

## 六、配置文件说明

### 6.1 application.yml 配置项

```yaml
upload:
  # 图片存储根目录（必须是已存在的目录）
  image-path: D:/项目图标
  # 支持的图片格式（逗号分隔）
  allowed-extensions: jpg,jpeg,png,gif,webp
  # 最大文件大小（单位：MB）
  max-file-size: 10
  # 图片访问基础URL（必须与静态资源映射一致）
  base-url: /images
```

### 6.2 关键配置说明

| 配置项 | 说明 | 注意事项 |
|-------|------|---------|
| image-path | 图片存储的根目录 | 必须确保目录存在且有写入权限 |
| allowed-extensions | 允许上传的文件扩展名 | 不区分大小写 |
| max-file-size | 单个文件最大大小（MB） | 需要与Spring Boot的文件上传限制配合 |
| base-url | 图片访问的基础URL | 必须与SaTokenConfig中的白名单和WebMvcConfig中的映射一致 |

## 七、安全性说明

### 7.1 文件安全验证

| 验证项 | 实现方式 | 说明 |
|-------|---------|------|
| 扩展名验证 | FileUtil.isAllowedExtension() | 只允许预定义的图片格式 |
| 文件大小验证 | ImageUploadServiceImpl | 限制最大上传大小为10MB |
| 空文件验证 | file.isEmpty() | 拒绝空文件上传 |

### 7.2 路径安全

| 安全措施 | 实现方式 | 说明 |
|---------|---------|------|
| 唯一文件名 | 时间戳+UUID | 避免文件名冲突和覆盖 |
| 不使用原始文件名 | 系统生成 | 防止路径穿越攻击 |
| 固定存储路径 | 配置化 | 避免任意路径写入 |

### 7.3 认证豁免

| 路径 | 豁免原因 |
|-----|---------|
| /api/upload/** | 上传接口需要对外公开 |
| /images/** | 图片访问需要对外公开 |

## 八、常见问题与解决方案

### 8.1 上传失败问题

#### 问题1：上传文件不能为空

**原因：** 未选择文件或文件为空

**解决：** 确保选择了有效的图片文件

#### 问题2：不支持的文件类型

**原因：** 上传了不支持的图片格式

**解决：** 使用支持的格式（jpg, jpeg, png, gif, webp）

#### 问题3：文件大小超过限制

**原因：** 上传的文件超过10MB

**解决：** 压缩图片或降低图片质量

#### 问题4：文件保存失败

**原因：** 磁盘空间不足或权限问题

**解决：** 检查D:/项目图标目录的写入权限

### 8.2 图片访问失败问题

#### 问题1：404 Not Found

**原因：** 图片文件不存在或路径错误

**解决：** 
1. 检查文件是否成功上传
2. 确认返回的URL路径正确
3. 检查文件是否被删除

#### 问题2：500 Internal Server Error

**原因：** 静态资源映射配置错误

**解决：** 
1. 检查 WebMvcConfig 中的 addResourceHandlers 配置
2. 确认 image-path 配置正确
3. 检查 base-url 是否一致

### 8.3 性能问题

#### 问题：上传响应慢

**可能原因：**
1. 网络带宽不足
2. 文件过大
3. 服务器磁盘IO瓶颈

**解决方案：**
1. 压缩图片后再上传
2. 使用分片上传（需要二次开发）
3. 升级服务器硬件

## 九、微信小程序集成示例

### 9.1 单个图片上传

```javascript
// pages/index/index.js
Page({
  data: {
    imageUrl: ''
  },

  // 选择并上传图片
  chooseAndUploadImage: function() {
    wx.chooseImage({
      count: 1,
      sizeType: ['compressed'],
      sourceType: ['album', 'camera'],
      success: (res) => {
        const tempFilePath = res.tempFilePaths[0];
        
        wx.uploadFile({
          url: 'http://localhost:8081/api/upload/image',
          filePath: tempFilePath,
          name: 'file',
          success: (uploadRes) => {
            const data = JSON.parse(uploadRes.data);
            
            if (data.code === 200) {
              const fullUrl = 'http://localhost:8081' + data.data.url;
              this.setData({
                imageUrl: fullUrl
              });
              
              wx.showToast({
                title: '上传成功',
                icon: 'success'
              });
              
              console.log('图片URL:', fullUrl);
              console.log('文件大小:', data.data.size);
              console.log('上传时间:', data.data.uploadTime);
            } else {
              wx.showToast({
                title: '上传失败：' + data.msg,
                icon: 'none'
              });
            }
          },
          fail: (err) => {
            console.error('上传失败:', err);
            wx.showToast({
              title: '上传失败',
              icon: 'none'
            });
          }
        });
      }
    });
  }
});
```

### 9.3 批量图片上传

```javascript
// pages/index/index.js
Page({
  data: {
    imageUrls: []
  },

  // 选择并批量上传图片
  chooseAndUploadImages: function() {
    wx.chooseImage({
      count: 9,
      sizeType: ['compressed'],
      sourceType: ['album', 'camera'],
      success: (res) => {
        const tempFilePaths = res.tempFilePaths;
        const uploadedUrls = [];
        let uploadCount = 0;
        
        wx.showLoading({
          title: '上传中...',
          mask: true
        });

        for (let i = 0; i < tempFilePaths.length; i++) {
          wx.uploadFile({
            url: 'http://localhost:8081/api/upload/image',
            filePath: tempFilePaths[i],
            name: 'file',
            success: (uploadRes) => {
              const data = JSON.parse(uploadRes.data);
              
              if (data.code === 200) {
                const fullUrl = 'http://localhost:8081' + data.data.url;
                uploadedUrls.push(fullUrl);
                uploadCount++;
                
                if (uploadCount === tempFilePaths.length) {
                  wx.hideLoading();
                  this.setData({
                    imageUrls: uploadedUrls
                  });
                  
                  wx.showToast({
                    title: '上传成功',
                    icon: 'success'
                  });
                  
                  console.log('所有图片URL:', uploadedUrls);
                }
              }
            },
            fail: (err) => {
              console.error('第' + (i + 1) + '张图片上传失败:', err);
              uploadCount++;
              
              if (uploadCount === tempFilePaths.length) {
                wx.hideLoading();
              }
            }
          });
        }
      }
    });
  }
});
```

## 十、部署说明

### 10.1 开发环境

1. 确保 D:/项目图标 目录存在
2. 启动后端服务（端口：8081）
3. 在微信开发者工具中勾选"不校验合法域名"
4. 测试上传和访问功能

### 10.2 生产环境部署

#### 步骤1：修改配置文件

在生产服务器上修改 `application.yml`：

```yaml
upload:
  image-path: /var/www/luxury-watch/images
  allowed-extensions: jpg,jpeg,png,gif,webp
  max-file-size: 10
  base-url: /images
```

#### 步骤2：创建存储目录

```bash
sudo mkdir -p /var/www/luxury-watch/images
sudo chmod 755 /var/www/luxury-watch/images
```

#### 步骤3：打包部署

```bash
mvn clean package -DskipTests
java -jar target/luxury-watch-backend.jar
```

#### 步骤4：配置反向代理（可选）

如果需要通过域名访问：

```nginx
server {
    listen 80;
    server_name api.luxurywatch.com;
    
    location /api/ {
        proxy_pass http://127.0.0.1:8081/api/;
    }
    
    location /images/ {
        alias /var/www/luxury-watch/images/;
        expires 30d;
    }
}
```

### 10.3 图片迁移

如果需要将开发环境的图片迁移到生产环境：

```bash
scp -r D:/项目图标/* user@server:/var/www/luxury-watch/images/
```

## 十一、接口调用流程图

```
┌─────────────┐
│  选择图片   │
└──────┬──────┘
       │
       ▼
┌─────────────┐
│  前端验证   │  格式、大小
└──────┬──────┘
       │
       ▼
┌─────────────┐
│  调用上传接口 │  POST /api/upload/image
└──────┬──────┘
       │
       ▼
┌─────────────┐
│  后端验证   │  文件类型、大小、空文件
└──────┬──────┘
       │
       ▼
┌─────────────┐
│  生成路径   │  年月文件夹 + 唯一文件名
└──────┬──────┘
       │
       ▼
┌─────────────┐
│  保存文件   │  D:/项目图标/年月/文件名
└──────┬──────┘
       │
       ▼
┌─────────────┐
│  返回URL   │  /images/年月/文件名
└──────┬──────┘
       │
       ▼
┌─────────────┐
│  前端使用   │  拼接完整URL显示图片
└─────────────┘
```

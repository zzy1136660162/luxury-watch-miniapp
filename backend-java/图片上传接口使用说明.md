# 图片上传接口使用说明

## 功能概述

本系统提供图片上传功能，支持将图片上传到本地文件系统（D:/项目图标），并通过HTTP接口访问。

## 已创建的文件

### 配置文件
- `application.yml` - 添加了图片上传配置
- `UploadConfig.java` - 图片上传配置属性类
- `WebMvcConfig.java` - 静态资源映射配置

### 业务代码
- `FileUtil.java` - 文件工具类
- `ImageUploadDTO.java` - 上传响应DTO
- `ImageUploadService.java` - 上传服务接口
- `ImageUploadServiceImpl.java` - 上传服务实现
- `ImageUploadController.java` - 上传控制器

## 接口列表

### 1. 单个图片上传

**接口地址：** `POST /api/upload/image`

**请求参数：**
- `file`: MultipartFile（图片文件）

**请求示例：**
```html
<form action="/api/upload/image" method="post" enctype="multipart/form-data">
    <input type="file" name="file" />
    <button type="submit">上传</button>
</form>
```

**成功响应：**
```json
{
    "code": 0,
    "message": "success",
    "data": {
        "url": "/images/2026-03/heroImage_1743401234567_abc123def.jpg",
        "filename": "heroImage_1743401234567_abc123def.jpg",
        "size": 102400,
        "uploadTime": "2026-03-31T14:30:00",
        "originalFilename": "heroImage.jpg",
        "fileType": "jpg"
    }
}
```

**错误响应：**
```json
{
    "code": 500,
    "message": "上传文件不能为空",
    "data": null
}
```

### 2. 批量图片上传

**接口地址：** `POST /api/upload/images`

**请求参数：**
- `files`: MultipartFile[]（多个图片文件）

**请求示例：**
```html
<form action="/api/upload/images" method="post" enctype="multipart/form-data">

```

**成功响应：**

### 3. 图片访问

**访问地址：** `/images/{year-month}/{filename}`

**示例：**
```
GET /images/2026-03/heroImage_1743401234567_abc123def.jpg
```

**说明：** 此接口通过静态资源映射自动实现，无需单独开发。

## 微信小程序端使用示例

### 1. 上传图片

```typescript
wx.chooseImage({
  count: 1,
  sizeType: ['compressed'],
  sourceType: ['album', 'camera'],
  success: (res) => {
    const tempFilePath = res.tempFilePaths[0];
    
    wx.uploadFile({
      url: 'http://localhost:9000/api/upload/image',
      filePath: tempFilePath,
      name: 'file',
      success: (uploadRes) => {
        const data = JSON.parse(uploadRes.data);
        if (data.code === 0) {
          const imageUrl = data.data.url;
          console.log('图片上传成功:', imageUrl);
          
          // 更新页面数据
          this.setData({
            heroImage: 'http://localhost:9000' + imageUrl
          });
          
          // 保存到服务器或数据库
          // this.saveImageToServer(imageUrl);
        } else {
          wx.showToast({
            title: '上传失败：' + data.message,
            icon: 'none'
          });
        }
      },
      fail: (err) => {
        console.error('上传失败:', err);
        wx.showToast({
          title: '上传失败',
          icon: 'none'
        });
      }
    });
  }
});
```

### 2. 在WXML中使用图片

```xml
<image src="{{heroImage}}" mode="aspectFill" />
```

### 3. 批量上传图片

```typescript
wx.chooseImage({
  count: 9,
  sizeType: ['compressed'],
  sourceType: ['album', 'camera'],
  success: (res) => {
    const tempFilePaths = res.tempFilePaths;
    const uploadedUrls = [];
    
    // 逐个上传
    let uploadedCount = 0;
    for (let i = 0; i < tempFilePaths.length; i++) {
      wx.uploadFile({
        url: 'http://localhost:9000/api/upload/image',
        filePath: tempFilePaths[i],
        name: 'file',
        success: (uploadRes) => {
          const data = JSON.parse(uploadRes.data);
          if (data.code === 0) {
            uploadedUrls.push(data.data.url);
            uploadedCount++;
            
            // 全部上传完成
            if (uploadedCount === tempFilePaths.length) {
              console.log('全部上传成功:', uploadedUrls);
              this.setData({
                images: uploadedUrls
              });
            }
          }
        }
      });
    }
  }
});
```

## 错误处理

### 常见错误

1. **上传文件不能为空**
   - 原因：未选择文件或文件为空
   - 解决：确保选择了有效的图片文件

2. **不支持的文件类型**
   - 原因：上传了不支持的图片格式
   - 解决：使用支持的格式（jpg, jpeg, png, gif, webp）

3. **文件大小超过限制**
   - 原因：上传的文件超过10MB
   - 解决：压缩图片或降低图片质量

4. **文件保存失败**
   - 原因：磁盘空间不足或权限问题
   - 解决：检查D:/项目图标目录的写入权限

## 注意事项

1. **路径配置**
   - 图片存储路径在 `application.yml` 中配置
   - 修改配置后需要重启服务

2. **目录创建**
   - 系统会自动创建年月子目录
   - 确保D:/项目图标目录存在且可写

3. **文件名生成**
   - 使用时间戳+UUID生成唯一文件名
   - 避免文件名冲突和覆盖

4. **安全考虑**
   - 已验证文件扩展名
   - 限制文件大小
   - 生成唯一文件名防止路径穿越

5. **跨域访问**
   - 后端已配置CORS
   - 小程序需在开发者工具中勾选"不校验合法域名"

## 测试步骤

1. 启动后端服务
2. 确保D:/项目图标目录存在
3. 使用Postman或浏览器测试上传接口
4. 在小程序中测试图片上传和显示

## 部署到服务器

1. 将图片上传接口部署到生产服务器
2. 修改 `application.yml` 中的 `image-path` 为服务器上的路径
3. 确保服务器上有对应的目录并有写入权限
4. 在小程序中更换接口地址为生产环境地址

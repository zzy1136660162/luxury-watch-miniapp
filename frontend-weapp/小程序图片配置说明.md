# 小程序图片配置说明

## 功能概述

为微信小程序配置了图片访问前缀，确保从后端API获取的商品图片能够正常显示。

## 实现内容

### 1. 图片配置文件

**文件位置：** `miniprogram/utils/config.ts`

**配置内容：**
```typescript
const imageConfig = {
  // 小程序专用图片访问前缀
  baseUrl: 'http://localhost:8081',
  
  // 图片路径前缀
  imagePath: '/images',
};
```

### 2. URL转换函数

提供了 `getFullImageUrl()` 函数，自动将相对路径拼接为完整URL：

```typescript
import { getFullImageUrl } from '../../utils/config';

// 示例
const relativePath = '/images/2026-03/1774941078635_68fb2686.jpg';
const fullUrl = getFullImageUrl(relativePath);
// 结果: http://localhost:8081/images/2026-03/1774941078635_68fb2686.jpg
```

## 配置修改

### 修改图片服务器地址

如果需要修改图片服务器地址，编辑 `miniprogram/utils/config.ts`：

```typescript
// 开发环境
const imageConfig = {
  baseUrl: 'http://localhost:8081',  // 本地开发服务器
  imagePath: '/images',
};

// 生产环境
const imageConfig = {
  baseUrl: 'https://your-production-server.com',  // 生产服务器
  imagePath: '/images',
};
```

### 添加新的图片处理

如果需要在其他页面添加图片路径处理：

1. **导入配置工具：**
```typescript
import { getFullImageUrl } from '../../utils/config';
```

2. **在 attached 生命周期中处理：**
```typescript
attached() {
  this.processImages();
},

methods: {
  processImages() {
    // 将相对路径转换为完整URL
    this.setData({
      myImage: getFullImageUrl(this.data.myImage),
      myImageList: this.data.myImageList.map(img => getFullImageUrl(img))
    });
  }
}
```

## 使用流程

### 从后端API获取数据

当从后端API获取商品数据时，后端返回的图片路径格式为：

```json
{
  "id": "1",
  "name": "曜石金·自动机械",
  "image": "/images/2026-03/1774941078635_68fb2686.jpg"
}
```

### 自动转换

在小程序的 `attached` 生命周期中，会自动调用处理方法，将相对路径转换为完整URL：

```
原始路径: /images/2026-03/1774941078635_68fb2686.jpg
              ↓
前缀拼接: http://localhost:8081
              ↓
完整URL: http://localhost:8081/images/2026-03/1774941078635_68fb2686.jpg
```

### 模板中使用

在 WXML 模板中直接使用，无需额外处理：

```xml
<image src="{{item.image}}" mode="aspectFill" />
```

小程序会自动使用转换后的完整URL加载图片。

## 注意事项

### 1. 网络权限

确保在小程序的 `app.json` 中配置了 `permission`，允许访问图片服务器：

```json
{
  "permission": {
    "scope.userLocation": {
      "desc": "您的位置信息将用于查找附近门店"
    }
  },
  "networkTimeout": {
    "request": 10000,
    "downloadFile": 10000
  }
}
```

### 2. 域名配置

在微信公众平台配置合法域名：
- 登录微信公众平台
- 进入「开发」→「开发设置」
- 在「服务器域名」中添加 `request` 合法域名：`http://localhost:8081`
- 添加 `downloadFile` 合法域名：`http://localhost:8081`

### 3. 开发环境

在微信开发者工具中，可以勾选「不校验合法域名」选项（仅限开发阶段）：

**设置 → 项目设置 → 勾选"不校验合法域名"**

### 4. 图片格式

支持的图片格式：
- JPEG（.jpg, .jpeg）
- PNG（.png）
- GIF（.gif）
- WebP（.webp）

## 常见问题

### Q: 图片无法显示怎么办？

**A:** 检查以下几点：
1. 确认后端服务已启动并运行在正确端口
2. 确认图片文件存在于服务器目录
3. 检查浏览器控制台是否有404错误
4. 确认小程序配置了正确的域名（开发阶段可勾选"不校验合法域名"）

### Q: 如何区分开发环境和生产环境？

**A:** 可以通过以下方式处理：
```typescript
const isDev = process.env.NODE_ENV === 'development';

const imageConfig = {
  baseUrl: isDev ? 'http://localhost:8081' : 'https://your-production-server.com',
  imagePath: '/images',
};
```

### Q: 图片URL已经是完整的还需要转换吗？

**A:** 不需要。`getFullImageUrl()` 函数会自动检测：
- 如果传入的是完整URL（以 `http://` 或 `https://` 开头），直接返回
- 如果传入的是相对路径，才会拼接前缀

## 技术支持

2. 后端服务是否正常运行
3. 图片文件是否存在
4. 网络连接是否正常

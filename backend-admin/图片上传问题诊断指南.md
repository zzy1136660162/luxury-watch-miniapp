# 图片上传问题诊断指南

## 快速诊断步骤

### 1. 查看上传配置（第一步）

上传组件初始化时会打印配置信息：

```
🌐 [ImageUpload] 上传配置:
  VITE_APP_API_BASEURL: http://localhost:8081
  VITE_OPEN_PROXY: true
  开发模式: true
  是否使用代理: true
  baseUrl: http://localhost:8081
  actualBaseUrl: /proxy
  uploadUrl: /proxy/api/upload/image
```

**检查点：**
- ✅ 如果 `VITE_OPEN_PROXY: true` 和 `是否使用代理: true`，则使用 `/proxy/api/upload/image`
- ❌ 如果配置不对，检查 `.env.development` 文件

### 2. 查看 Network 面板（最重要）

在上传图片时，打开 **Network 面板**：

1. 按 F12 打开开发者工具
2. 点击 **Network** 标签
3. 勾选 **Fetch/XHR** 筛选
4. 上传图片
5. 查找 `upload` 相关的请求

**检查请求：**
- ✅ 请求 URL 应该是：`/proxy/api/upload/image` 或 `http://localhost:8081/api/upload/image`
- ✅ 请求 Method 应该是：`POST`
- ✅ 响应状态码应该是：`200`

### 3. 查看响应内容

在 Network 面板中，点击上传请求，查看 **Response** 标签：

**正确的响应：**
```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "url": "/images/2026-03/xxx.jpg",
    ...
  }
}
```

### 4. 查看控制台调试信息

上传成功后，查看控制台输出：

#### 如果成功：
```
🌐 [ImageUpload] 上传配置: {...}
📤 [ImageUpload] 开始上传文件: xxx.jpg 大小: xxx 类型: xxx
✅ [ImageUpload] 文件验证通过，准备上传
🔍 [ImageUpload] ====== 上传成功回调触发 ======
📦 [ImageUpload] 原始 response 完整对象: {...}
📋 [ImageUpload] ====== response 详细结构 ======
1. response.status: 200
2. response.statusText: OK
3. response.data: {code: 200, data: {...}}
4. response.data.code: 200
5. response.data.data: {url: "/images/..."}
6. response.data.data.url: /images/...
🎯 [ImageUpload] ====== 解析后的 resData ======
resData: {code: 200, data: {...}}
resData?.code: 200
resData?.data?.url: /images/...
✅ [ImageUpload] ====== 上传成功 ======
✅ 图片URL: /images/...
========================================
```

#### 如果失败：
```
❌ [ImageUpload] 上传失败，错误信息: xxx
```

## 常见问题

### ❌ 问题1：请求没有发出去

**现象：** Network 面板看不到上传请求

**检查：**
1. 检查 `VITE_OPEN_PROXY` 是否为 `true`
2. 检查 `uploadUrl` 配置是否正确
3. 检查是否正确引入了组件

### ❌ 问题2：请求发出去但返回404

**现象：** Network 显示 404

**检查：**
1. 检查后端服务是否启动在 `8081` 端口
2. 检查上传接口路径是否正确：`/api/upload/image`
3. 检查后端日志

### ❌ 问题3：response.data 是 undefined

**现象：** 控制台显示 `response.data: undefined`

**原因：** 请求可能没有经过 axios 拦截器，或者响应格式不对

**检查：**
1. 在 Network 面板查看实际的响应内容
2. 检查 API 拦截器配置
3. 检查是否使用了代理

### ❌ 问题4：response.data.code 不是 200

**现象：** `resData.code` 不是 200

**检查：**
1. 查看 `resData.msg` 获取错误信息
2. 检查后端日志
3. 检查权限配置

### ❌ 问题5：一直 loading 状态

**现象：** 上传后一直显示 loading，不显示图片

**可能原因：**
1. `handleSuccess` 没有被调用（可能是网络错误）
2. 响应数据结构不对
3. 代码逻辑错误

**检查：**
1. 查看控制台是否有错误信息
2. 查看 Network 面板请求状态
3. 检查是否进入了 `handleError` 而不是 `handleSuccess`

## 调试信息清单

| 阶段 | 输出内容 | 作用 |
|-----|---------|------|
| 组件初始化 | 配置信息、uploadUrl | 确认请求地址正确 |
| 文件验证 | 文件信息 | 确认文件符合要求 |
| 上传成功 | response 完整结构 | 确认响应数据 |
| URL 计算 | 计算过程 | 确认图片路径 |

## 需要提供的信息

如果以上方法都无法解决问题，请提供：

1. **完整的控制台输出**（从组件初始化到上传结束的所有日志）
2. **Network 面板截图**（包含请求和响应）
3. **后端日志**（如果有）

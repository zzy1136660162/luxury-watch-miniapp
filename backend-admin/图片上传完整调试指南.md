# 图片上传完整调试指南

## 调试流程

### 步骤1：重启前端服务

确保最新的代码已经加载：
```bash
cd backend-admin
npm run dev
```

### 步骤2：打开浏览器开发者工具

1. 按 `F12` 打开开发者工具
2. 点击 **Console** 标签
3. 点击 **Network** 标签
4. 勾选 **Fetch/XHR** 筛选器

### 步骤3：执行上传并观察

在商品管理页面，点击新增或编辑，上传一张图片。

**常见错误码：**
- `400` - 请求参数错误
- `401` - 未授权
- `403` - 禁止访问
- `404` - 接口不存在
- `500` - 服务器内部错误

### 如果没有任何日志

**检查：**
1. 是否打开了正确的页面
2. 是否点击了上传按钮
3. 是否选择了文件

## Network 面板调试

### 查看上传请求

1. 在 Network 面板中查找 `upload` 请求
2. 点击该请求查看详情

### 请求标签

**检查项：**
- ✅ Request URL: `/proxy/api/upload/image` 或 `http://localhost:8081/api/upload/image`
- ✅ Request Method: `POST`
- ✅ Request Headers: Content-Type 应该是 `multipart/form-data`

### 响应标签

**检查项：**
- ✅ Status: `200 OK` 或 `4xx/5xx`
- ✅ Response: JSON 格式的响应数据

**如果 Status 是 4xx/5xx：**
- 查看后端日志
- 检查权限配置
- 检查接口路径

## 快速检查清单

| 检查项 | 正常状态 | 异常状态 |
|-------|---------|---------|
| 文件选择 | 显示文件信息 | 无日志 |
| beforeUpload | 返回 true | 显示错误信息 |
| handleSuccess | 触发 | 不触发 |
| handleError | 不触发 | 触发 |
| 请求 Status | 200 | 4xx/5xx |

## 需要提供的信息

如果以上方法都无法解决问题，请提供：

1. **完整的控制台输出**（从文件选择到结束的**所有**日志）
2. **Network 面板截图**（包含请求和响应）
3. **后端控制台日志**（如果有）

## 常见问题

### ❌ 问题1：beforeUpload 返回 false

**原因：** 文件类型或大小不符合要求

**解决：** 上传 JPG/PNG/GIF/WebP 格式且小于 10MB 的图片

### ❌ 问题2：handleError 被触发

**原因：** 请求失败（网络错误、服务器错误等）

**解决：** 查看错误信息，检查后端服务

### ❌ 问题3：没有任何日志

**原因：** 可能没有选择文件或没有点击上传

**解决：** 确保选择了文件并点击了上传

### ❌ 问题4：一直 loading

**原因：**
1. 请求未完成
2. 请求失败但未触发 handleError

**解决：** 查看 Network 面板请求状态

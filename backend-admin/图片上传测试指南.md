# 后端管理员平台图片上传测试指南

## 测试前准备

### 1. 确保环境就绪

- [ ] 后端服务已启动（端口：8081）
- [ ] 前端开发服务器已启动（端口：5173）
- [ ] `D:/项目图标` 目录存在且可写
- [ ] 准备测试图片文件（建议准备1-2张测试图片）

### 2. 检查配置

#### 后端配置检查

文件：`backend-java/src/main/resources/application.yml`

```yaml
upload:
  image-path: D:/项目图标
  allowed-extensions: jpg,jpeg,png,gif,webp
  max-file-size: 10
  base-url: /images
```

#### 前端配置检查

文件：`backend-admin/.env.development`

```env
VITE_APP_API_BASEURL = http://localhost:8081
```

#### SaToken配置检查

文件：`backend-java/src/main/java/com/luxurywatch/config/SaTokenConfig.java`

确保以下路径在豁免列表中：
- `/api/upload/**`
- `/images/**`

## 测试步骤

### 测试1：单个图片上传接口（Postman）

1. **打开Postman**
2. **创建新请求**
   - 方法：`POST`
   - URL：`http://localhost:8081/api/upload/image`

3. **设置请求体**
   - 点击 Body
   - 选择 form-data
   - KEY: `file`
   - VALUE: 选择一张测试图片
   - VALUE类型: File

4. **发送请求**
   - 点击 Send 按钮

5. **验证响应**
   - 应收到状态码 200
   - 响应体应包含：
     ```json
     {
         "code": 200,
         "msg": "success",
         "data": {
             "url": "/images/2026-03/test_xxx.jpg",
             "filename": "test_xxx.jpg",
             "size": 102400,
             "uploadTime": "2026-03-31T14:30:00",
             "originalFilename": "test.jpg",
             "fileType": "jpg"
         }
     }
     ```

6. **验证文件存储**
   - 打开 `D:/项目图标/2026-03/`
   - 确认图片文件已保存

### 测试2：图片访问接口（浏览器）

1. **使用上传返回的URL**
   - 复制上传响应中的 `url` 字段
   - 例如：`/images/2026-03/test_xxx.jpg`

2. **拼接完整URL**
   - `http://localhost:8081/images/2026-03/test_xxx.jpg`

3. **在浏览器中访问**
   - 打开浏览器
   - 访问上述URL
   - 应能正常显示图片

### 测试3：后端管理员平台上传功能

#### 步骤1：登录管理平台

1. 打开浏览器
2. 访问：`http://localhost:5173`
3. 使用管理员账号登录

#### 步骤2：进入商品管理

1. 在左侧菜单找到"商品管理"
2. 点击进入商品列表页面

#### 步骤3：新增商品并上传图片

1. 点击"新增商品"按钮
2. 填写商品信息：
   - 商品名称：`测试商品001`
   - 商品编码：`TEST001`
   - 商品分类：选择一个分类（如"经典"）
   - 商品价格：`999.00`
   - 库存数量：`100`

3. **上传商品图片**：
   - 找到"商品图片"区域
   - 点击上传区域（虚线框）
   - 选择一张测试图片
   - 等待上传完成

4. **验证上传结果**：
   - 上传成功后，图片应显示在上传区域
   - 检查浏览器控制台，应有类似日志：
     ```
     [ImageUpload] 上传成功: /images/2026-03/test_xxx.jpg
     ```

5. **填写商品描述**：
   - 描述：`这是一款测试商品`

6. **提交表单**：
   - 点击"确定"按钮
   - 应收到成功提示："新增成功"
   - 商品列表中应能看到新商品
   - 商品图片应正确显示

#### 步骤4：编辑商品并更换图片

1. 在商品列表中找到刚添加的商品
2. 点击"编辑"按钮
3. 在表单中可以看到已上传的图片
4. 点击图片区域，重新上传一张新图片
5. 确认图片已更新
6. 点击"确定"保存修改
7. 返回商品列表，确认图片已更换

## 常见问题排查

### 问题1：上传按钮点击无反应

**可能原因：**
- 后端服务未启动
- 前端环境变量配置错误

**排查步骤：**
1. 检查后端是否运行：`netstat -an | grep 8081`
2. 检查前端控制台是否有错误
3. 确认 `VITE_APP_API_BASEURL` 配置正确

### 问题2：上传提示成功但图片不显示

**可能原因：**
- 图片路径保存错误
- 图片未真正上传到服务器
- 静态资源映射未生效

**排查步骤：**
1. 检查浏览器Network面板，查看上传请求响应
2. 检查 `D:/项目图标` 目录是否有文件
3. 检查后端控制台日志
4. 确认 `WebMvcConfig` 配置正确

### 问题3：图片上传接口返回500错误

**可能原因：**
- 服务器存储目录不存在
- 目录无写入权限
- 磁盘空间不足

**排查步骤：**
1. 检查 `D:/项目图标` 目录是否存在
2. 检查目录权限
3. 检查磁盘空间
4. 查看后端错误日志

### 问题4：CORS跨域错误

**可能原因：**
- 后端CORS配置未生效
- 浏览器发送了预检请求

**排查步骤：**
1. 检查浏览器控制台错误信息
2. 确认 `WebMvcConfig` 中的 `addCorsMappings` 配置
3. 检查请求头是否正确

## 测试数据准备

### 推荐测试图片

1. **小图片（< 1MB）**
   - 用于测试正常上传
   - 文件名：`test_small.jpg`

2. **中等图片（1-5MB）**
   - 用于测试边界情况
   - 文件名：`test_medium.jpg`

3. **接近限制的图片（9-10MB）**
   - 用于测试大小限制
   - 文件名：`test_large.jpg`

4. **超过限制的图片（> 10MB）**
   - 用于测试错误处理
   - 文件名：`test_oversize.jpg`

5. **其他格式图片**
   - PNG格式：`test.png`
   - GIF格式：`test.gif`
   - WebP格式：`test.webp`

6. **不支持的格式**
   - TXT文件：`test.txt`
   - PDF文件：`test.pdf`

### 测试数据生成脚本（可选）

如果需要生成测试图片，可以使用以下Python脚本：

```python
from PIL import Image
import os

def create_test_image(filename, width, height, size_mb=None):
    img = Image.new('RGB', (width, height), color=(73, 109, 137))
    img.save(filename, quality=95)

    if size_mb:
        current_size = os.path.getsize(filename) / (1024 * 1024)
        if current_size < size_mb:
            print(f"Created {filename} with size {current_size:.2f}MB")
        else:
            print(f"Warning: {filename} is {current_size:.2f}MB, target was {size_mb}MB")

# 创建测试图片
os.makedirs('test_images', exist_ok=True)
os.chdir('test_images')

create_test_image('test_small.jpg', 400, 400)
create_test_image('test_medium.jpg', 800, 800)
print("Test images created successfully!")
```
